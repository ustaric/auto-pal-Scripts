# 팰월드 통합 관리 시스템 설치 및 운영 가이드

이 가이드는 구글 클라우드(GCP) VM 인스턴스를 생성하기 위한 SSH 키 발급 과정부터, 가상 서버에 처음 접속하여 게임 서버와 디스코드 관리 봇을 빌드하고 안정적으로 상시 구동하기까지의 전 과정을 다룹니다.

---

## 📋 시스템 구성 및 주요 기능
* **대상 시스템**: Google Cloud Platform (GCP)
* **권장 가상 서버 OS**: Ubuntu 22.04 LTS
* **기본 작업 디렉토리**: `/home/<사용자_계정명>/palworld`
* **주요 기능 요약**:
  - **이중 동적 자동 백업**: 유저 접속률이 높은 피크타임(평일 19시~02시, 주말 09시~02시)에는 30분 간격, 비피크타임에는 1시간 간격으로 동적 자동 백업을 수행합니다. (접속자가 0명일 시 불필요한 백업을 차단하고 대기)
  - **초고속 실시간 감지**: 유저 수 변동 발생 시 8초 이내에 현황판을 강제 즉시 동기화합니다. (무변동 시에는 5분 주기로 디스코드 수정을 제한하여 서버 자원 극대화 보존)
  - **수동 백업 슬래시 명령어**: 관리자가 디스코드 창에서 `/backup`을 입력하면 즉시 세이브를 강제 저장 및 압축 백업합니다.
  - **정제된 대시보드 UI**: 기가바이트 단위를 직관적으로 표기하는 RAM 인터페이스(`2.1GB/16GB`) 및 모바일 화면에서도 깨지지 않는 가로폭 보존형 레이아웃을 제공합니다.
  - **전용 알림 시스템**: 백업 성공/실패 여부 및 정기 점검 재시작 상황을 디스코드 로그 전용 채널(`LOG_CHANNEL_ID`)로 실시간 포워딩합니다.

## 🔒 보안 및 안전성 안내 (Security & Safety)

본 시스템은 외부 해킹 공격 및 허가되지 않은 사용자의 서버 오용을 방지하기 위해 다음과 같은 **다중 보안 설계**가 적용되어 있습니다.

### 1. 폐쇄형 내부망 통신 (Localhost Isolation)
* 봇(`main.py`)과 게임 서버(`docker-compose`)는 **동일한 GCP 가상 서버 내부**에 상주합니다.
* 봇이 정보를 수집하는 REST API 포트(`8212`)와 원격 명령어 포트(`25575`)는 외부 인터넷 IP 주소가 아닌 **로컬 루프백 주소(`127.0.0.1`)**를 통해서만 내부적으로 연결됩니다.

### 2. 가상 서버 방화벽 최소화 (Port Protection)
* 게임 플레이를 위해 열어두어야 하는 **게임 포트(`UDP 8211`) 외의 모든 제어 포트는 외부로부터 차단**할 수 있습니다.
* 구글 클라우드 방화벽 설정 시, RCON 포트(`25575`) 및 REST API 포트(`8212`)를 외부에 개방하지 않아도 내부 통신망을 통해 봇이 완벽하게 정보를 제어하므로 외부로부터의 불필요한 포트 노출을 방지합니다.

### 3. 디스코드 관리자 ID 검증 구조 (Role Verification)
* 서버 제어 및 백업과 관련된 중요 슬래시 명령어는 호출한 디스코드 유저의 숫자 고유 ID와 `env`에 지정된 **`ADMIN_IDS` 목록을 1대1로 대조하여 검증**합니다.
* 관리자로 등록되지 않은 일반 유저나 타인이 명령어를 오용하려 하더라도 봇이 즉각 거부(`❌ 권한이 없습니다.`)하도록 구현되어 있습니다.

## 🔑 [사전 준비] GCP 계정 생성 및 SSH 키 등록 가이드

GCP VM 가상 서버에 안전하게 접속하기 위해서는 SSH 비공개 키(`.ppk`)와 공개 키를 생성하여 등록해야 합니다.

### 1. MobaKeyGen (또는 puttygen.exe)으로 키 생성
1. MobaXterm 상단 메뉴에서 **Tools** > **MobaKeyGen (SSH Key Generator)**을 실행합니다. (독립 실행형 PuTTYgen 프로그램을 사용하셔도 동일합니다.)
2. 하단의 Parameters 항목에서 **RSA**를 선택하고, **Generate** 버튼을 클릭합니다.
3. 진행 표시줄이 차오르는 동안 마우스를 무작위로 계속 움직여 임의의 난수를 생성합니다.
4. 키 생성이 완료되면 화면 중간의 **Key comment** 부분을 원하는 **`<사용자_계정명>`**으로 변경합니다. (예: `gcp-user` 등 영어 소문자로 구성된 계정명 권장)
5. **Save private key** 버튼을 클릭하여 안전한 로컬 폴더에 비공개 키 파일(`.ppk`)을 저장합니다.
6. 맨 위 회색 텍스트 상자(`Public key for pasting into OpenSSH...`) 안에 있는 긴 문자열 전체를 복사합니다.

### 2. 구글 클라우드 콘솔에 공개 키 등록
1. 구글 클라우드 콘솔의 **Compute Engine** > **VM 인스턴스** 메뉴로 이동합니다.
2. 생성한 가상 서버 인스턴스의 이름을 클릭해 상세 페이지로 들어간 뒤, 상단의 **수정** 버튼을 클릭합니다.
3. 화면을 아래로 내려 **SSH 키** 항목을 찾은 후 **항목 추가**를 누릅니다.
4. 위에서 복사한 공개 키 전체를 입력란에 그대로 붙여넣습니다. (정상 등록 시 끝에 입력한 계정명이 감지됩니다.)
5. 맨 아래로 내려 **저장**을 누릅니다.

### 3. MobaXterm 세션 생성 및 최초 접속
1. MobaXterm 왼쪽 위 **Session** 버튼을 누르고 **SSH** 세션을 추가합니다.
2. **Remote host**: GCP에서 할당된 가상 서버의 **외부 IP 주소**를 입력합니다.
3. **Specify username**: 키 생성 시 Key comment에 입력했던 **`<사용자_계정명>`**을 그대로 입력합니다.
4. **Advanced SSH settings** 탭을 클릭한 뒤, **Use private key** 항목을 체크하고 저장해 두었던 비공개 키 파일(`.ppk`)을 등록합니다.
5. **OK**를 누르면 가상 서버에 암호 입력 없이 안전하게 최초 접속됩니다.

## 🛠️ 단계별 구축 가이드

### [단계 1] 최초 접속 및 권한 최적화 폴더 생성
GCP 터미널(`사용자@호스트명:~$`)에 접속하자마자 실행합니다. 파일 업로드 및 도커 구동 과정에서 권한 충돌이 일어나지 않도록 폴더 소유권을 사전에 지정하는 과정입니다.

```bash
# 1. 서버 구동용 폴더 생성
mkdir -p ~/palworld

# 2. 폴더 소유권을 현재 계정으로 확실하게 선언
sudo chown -R $USER:$USER ~/palworld

# 3. 폴더 권한 부여 (MobaXterm 업로드 및 가상환경 구동을 위한 최적 권한)
chmod 755 ~/palworld

# 4. 생성한 폴더 안으로 이동
cd ~/palworld
```

---

### [단계 2] 프로젝트 파일 구성 (PC에서 생성 후 업로드)
MobaXterm 왼쪽 파일 탐색 패널을 새로고침하여 생성된 `/home/<사용자_계정명>/palworld` 폴더로 진입한 후, 준비된 아래의 총 **5개 파일**을 드래그 앤 드롭으로 업로드합니다.

1. **`setup.sh`**
2. **`main.py`**
3. **`docker-compose.yml`**
4. **`.env`** (점이 들어간 파일 - 게임 내부 상세 규칙 설정용)
5. **`env`** (점이 없는 파일 - 디스코드 봇 설정용)

#### 💡 `env` 설정 파일 구성 양식 (점이 없는 파일)
`env` 파일은 디스코드 봇 및 경로 매핑을 위한 파일입니다. 아래 포맷 예시에 맞추어 정보를 입력하여 업로드하십시오.

```env
# --- 디스코드 설정 ---
DISCORD_TOKEN=발급받은_디스코드_봇_토큰
ADMIN_IDS=관리자_디스코드_숫자_ID (여러 명일 시 콤마로 구분 예: ID1,ID2)
CHANNEL_ID=현황판_출력용_디스코드_채널_숫자_ID
LOG_CHANNEL_ID=백업성공/재시작_로그_수신용_채널_숫자_ID

# --- 서버 정보 ---
SERVER_NAME=Palworld Korea
SERVER_IP=자신의_GCP_외부_IP_주소
SERVER_PORT=8211

# --- 게임 내부 설정 ---
ADMIN_PASSWORD=원하는_RCON_비밀번호 (예: 12341234)
RCON_PORT=25575
REST_API_PORT=8212

# --- 경로 설정 (<사용자_계정명> 자리에는 본인의 실제 계정명을 입력하십시오) ---
BASE_PATH=/home/<사용자_계정명>/palworld
SAVE_DATA_PATH=/home/<사용자_계정명>/palworld/palworld/Pal/Saved
BACKUP_PATH=/home/<사용자_계정명>/palworld/backup

# --- 운영 정책 ---
BACKUP_RETENTION_DAYS=2
MEMORY_RESTART_THRESHOLD=80
RESTART_TIME=04:00
```

### [단계 3] 자동 빌드 스크립트 실행
업로드가 완료되었다면 터미널 창에서 윈도우 스타일 줄바꿈 문자 정화 작업을 거친 뒤 설치 스크립트를 원클릭 구동합니다.

```bash
# 1. 윈도우 텍스트 편집기로 인한 에러(CRLF) 사전 자동 방지
sed -i 's/\r$//' setup.sh env .env

# 2. setup.sh 스크립트에 실행 권한 부여
chmod +x setup.sh

# 3. 자동 설치 스크립트 실행
./setup.sh
```

> **주의**: 팰월드 전용 서버 이미지를 최초로 다운로드하는 과정에서 약 **180초(3분)** 동안 대기 상태가 지속됩니다. 터미널 창을 닫지 마시고 설치 완료 메시지가 출력될 때까지 기다려 주십시오.

### [단계 4] 봇 계정에 Docker 권한 영구 등록 (필수)
설치 스크립트가 완전히 끝나면, 백그라운드에서 상시 가동 중인 디스코드 봇이 `sudo` 없이도 도커 백업 명령어(`docker exec`)를 안전하게 지시할 수 있도록 권한을 보완합니다.

```bash
# 1. 실제 사용자를 도커 제어 그룹에 영구 등록 (<사용자_계정명> 자리에는 본인의 실제 계정명을 입력하십시오)
sudo usermod -aG docker <사용자_계정명>

# 2. 변경된 권한 세션을 터미널에 즉시 적용
newgrp docker

# 3. 권한 변경 사항을 반영하기 위해 디스코드 봇 서비스 재시작
sudo systemctl restart palbot.service
```

## 📈 실시간 운영 및 모니터링 명령어 가이드

### 1. 봇 서비스 관련 제어 명령어
디스코드 봇은 가상 서버가 켜질 때 자동으로 같이 실행되는 시스템 서비스로 등록되어 있습니다.
* **봇 상태 확인** (작동 여부, 가동 시간 등 확인):
  ```bash
  sudo systemctl status palbot.service
  ```
* **봇 재시작** (코드가 수정되었거나 `env` 설정값이 바뀐 경우):
  ```bash
  sudo systemctl restart palbot.service
  ```

### 2. 봇 로그 실시간 추적 (모니터링)
정기 백업 알림, 8초 주기 접속자 변경 감지 로그, 예외 발생 에러 내역을 터미널에서 모니터링할 수 있습니다.
* **실시간 로그 지켜보기** (종료하려면 `Ctrl + C`):
  ```bash
  tail -f /home/<사용자_계정명>/palworld/bot_output.log
  ```

### 3. 게임 서버 도커 컨테이너 제어 명령어
* **현재 켜져 있는 도커 컨테이너 목록 확인**:
  ```bash
  docker ps
  ```
* **게임 서버 강제 정지** (설정 변경 시 등):
  ```bash
  cd ~/palworld && docker-compose down
  ```
* **게임 서버 다시 시작**:
  ```bash
  cd ~/palworld && docker-compose up -d
  ```

## 🎯 유저용 디스코드 명령어 가이드

봇을 서버에 초대할 때 **`bot`**과 **`applications.commands`** 권한을 모두 체크하고 초대했다면 디스코드 대화창에서 슬래시( `/` )를 입력해 다음 관리 도구를 호출할 수 있습니다.

1. **`/backup`**
   - 즉시 팰월드 게임에 저장을 명령하고, 가상 서버 로컬 드라이브에 안전하게 세이브 파일을 타르 압축하여 수동 백업을 생성합니다.
2. **`/clean`**
   - 현황판 전용 채널에서 고정된 상태판 메시지 **하나만 제외하고** 유저들의 잡다한 채팅과 시스템 메시지 목록들을 일괄 정리합니다. (사용 전, 보존할 상태판 메시지를 디스코드에서 미리 **고정**해 주셔야 지워지지 않습니다.)
3. **`/restart`**
   - 게임 내에 셧다운 방송을 송출하고 세이브를 저장한 뒤, 도커 컨테이너를 안전하게 리스타트합니다.
4. **`/stop` / `/start`**
   - 게임 서버를 디스코드 상에서 자유롭게 끄고 켤 수 있습니다. (관리자 전용)
