# 팰월드 통합 관리 시스템 설치 및 운영 가이드

이 가이드는 구글 클라우드(GCP) VM 인스턴스를 생성하기 위한 사전 준비(계정, 디스코드 봇, SSH 키 발급) 과정부터, 가상 서버에 처음 접속하여 게임 서버와 디스코드 관리 봇을 빌드하고 안정적으로 상시 구동하기까지의 전 과정을 다룹니다.

---

## 📋 시스템 구성 및 주요 개선 기능

*   **대상 시스템**: Google Cloud Platform (GCP)
*   **권장 가상 서버 OS**: Ubuntu 22.04 LTS
*   **기본 작업 디렉토리**: `/home/<사용자_계정명>/palworld` (환경 변수 `BASE_PATH`로 지정)

### 주요 개선 및 기능 요약

*   **환경 설정 일괄 변경 및 이중 승인 (`/설정변경`)**
    *   여러 환경 변수(`.env`) 값을 쉼표(`,`)로 구분하여 한 번에 변경합니다.
    *   입력된 키의 유효성을 검증하여 오타/누락 시 수정을 거부합니다.
    *   변경 전후 값을 비교하는 임베드와 [변경 승인] 버튼을 통해 안전한 이중 확인 절차를 제공합니다.
    *   설정 변경 공지를 포함한 모든 주요 시스템 알림 메시지는 **15분(900초) 후 자동 삭제**되어 채널 청결을 유지합니다.
*   **서버 재시작 시 안전 백업 연동 (`/restart`)**
    *   관리자가 `/restart` 명령 실행 시, 서버 정지 직전 데이터를 안전하게 저장하고 백업(`run_backup`)을 선행한 후 컨테이너를 재기동합니다.
*   **Docker Compose V2 호환성 및 안정성 확보**
    *   최신 도커 API 규격에 맞춰 Docker Compose V2 환경과의 호환성을 확보하고, 기존 구형 컴포즈의 크래시 오류를 해결했습니다.
    *   안전한 `subprocess.run` 호출 방식을 사용하여 쉘 인젝션 위험을 방지하고 경로 제어를 강화했습니다.
*   **지능형 상태 검증 (STARTING ➔ ONLINE)**
    *   도커 컨테이너가 켜지더라도 실제 게임 REST API가 버전 정보를 반환하기 전까지는 대시보드 상태를 **STARTING (로딩 중)**으로 고정합니다.
    *   `player_watcher_task`의 5초 주기 감시를 통해 로딩 완료 즉시 상태를 **ONLINE**으로 신속하게 업데이트합니다.
*   **지능형 퇴장 감시 예약 백업**
    *   동접자 0명 감지 시 15분(`EMPTY_BACKUP_DELAY`)간 대기 후 백업을 예약합니다.
    *   이 대기 시간 중 유저가 재접속하면 예약된 백업을 자동으로 취소하며, 15분간 0명 유지 시 최종 세이브 데이터 보호를 위해 백업을 실행합니다.
*   **12시간 주기 디스크 잔량 모니터링 및 경고**
    *   `scheduled_disk_check_task`를 통해 12시간마다 디스크 사용률을 체크합니다.
    *   사용률이 **80% 이상**일 경우 지정된 로그 채널로 경고 임베드 메시지를 전송하여 용량 관리를 돕습니다.
*   **실시간 디스크 상태 조회 (`/디스크`)**
    *   12시간 대기 없이, `/디스크` 명령어로 현재 서버의 디스크 전체 용량, 사용량, 사용률을 실시간 임베드로 즉시 조회할 수 있습니다.
*   **향상된 이중 동적 자동 백업**
    *   유저 접속률이 높은 피크타임(평일 19시~02시, 주말 09시~02시)에는 30분 간격, 비피크타임에는 1시간 간격으로 동적 자동 백업을 수행합니다. (접속자가 0명일 시 백업 파일 생성을 대기하여 시스템 리소스를 보존합니다.)
*   **정제된 대시보드 UI**
    *   기가바이트 단위를 직관적으로 표기하는 RAM 인터페이스(`2.1GB/16GB`) 및 모바일 화면에서도 가로폭이 보존되도록 레이아웃을 구성했습니다.
*   **전용 알림 시스템**
    *   백업 성공/실패, 정기 점검 재시작, 디스크 용량 경고 등의 상황을 디스코드 로그 전용 채널(`LOG_CHANNEL_ID`)로 실시간 포워딩합니다.

## 🔒 보안 및 안전성 안내 (Security & Safety)

본 시스템은 외부 해킹 공격 및 허가되지 않은 사용자의 서버 오용을 방지하기 위해 다음과 같은 **다중 보안 설계**가 적용되어 있습니다.

### 1. 폐쇄형 내부망 통신 (Localhost Isolation)
*   봇(`main.py`)과 게임 서버(`docker-compose`)는 **동일한 GCP 가상 서버 내부**에 상주합니다.
*   봇이 정보를 수집하는 REST API 포트(`8212`)와 원격 명령어 포트(`25575`)는 외부 인터넷 IP 주소가 아닌 **로컬 루프백 주소(`127.0.0.1`)**를 통해서만 내부적으로 연결됩니다.

### 2. 가상 서버 방화벽 최소화 (Port Protection)
*   게임 플레이를 위해 열어두어야 하는 **게임 포트(`UDP 8211`) 외의 모든 제어 포트는 외부로부터 차단**할 수 있습니다.
*   구글 클라우드 방화벽 설정 시, RCON 포트(`25575`) 및 REST API 포트(`8212`)를 외부에 개방하지 않아도 내부 통신망을 통해 봇이 직접 정보를 제어하므로 외부로부터의 불필요한 포트 노출을 방지합니다.

### 3. 디스코드 관리자 ID 검증 구조 (Role Verification)
*   서버 제어 및 백업과 관련된 중요 슬래시 명령어는 호출한 디스코드 유저의 숫자 고유 ID와 `env`에 지정된 **`ADMIN_IDS` 목록을 1대1로 대조하여 검증**합니다.
*   관리자로 등록되지 않은 일반 유저나 타인이 명령어를 사용하려 하더라도 봇이 즉각 거부(`❌ 권한이 없습니다.`)하도록 구현되어 있습니다.

### 4. 강화된 환경 변수 사전 검증 (Fail-Fast)
*   봇 기동 직후 `env` 및 `.env` 파일의 필수 설정값(토큰, 채널 ID, 경로, 포트 범위 등)을 검증합니다.
*   유효하지 않은 값이 발견될 경우, 봇이 구동하기 전에 스스로를 안전하게 중단하여 잠재적 오류나 오작동을 사전에 차단합니다.

### 5. 안전한 Subprocess 실행 및 쉘 인젝션 방지
*   쉘 주입 취약점을 사전에 차단하기 위해 `os.system()` 대신 `subprocess.run()`에 리스트 인자 형식(`["docker", "exec", ...]`)을 사용합니다.
*   디렉토리 이동(`cd ... && docker-compose ...`) 방식 대신 `cwd=BASE_PATH` 옵션을 활용하여 파이썬이 리눅스 디스크 디렉토리를 직접 가리키도록 정밀하게 지정했습니다.

### 6. 고급 로깅 및 예외 추적 (Traceback)
*   `logging` 라이브러리의 `RotatingFileHandler`를 적용하여 `palworld_bot.log` 파일에 로그를 기록합니다. 파일 크기가 5MB에 도달하면 새 파일을 열고 기존 로그를 보존(최대 3개)하여 디스크 용량 오버플로우를 예방합니다.
*   모든 `print()` 구문을 `logger.info()`, `logger.error()`, `logger.warning()` 등으로 교체했습니다.
*   의미 불명확한 `except:` 처리 대신 `except Exception as e:`와 `logger.exception()`을 사용하여 예외 발생 시 상세한 에러 로그와 호출 스택(`traceback`)을 기록하도록 정비했습니다.

## 🔑 [사전 준비] 개발 환경 및 클라우드 사전 세팅 가이드

팰월드 서버를 원활히 구동하고 관리하기 위해 가상 서버 생성 이전에 아래의 세팅을 순서대로 진행해야 합니다.

### 1. GCP 무료 티어 계정 가입 및 VM 인스턴스 준비
1.  **구글 클라우드 플랫폼(GCP)**에 접속하여 신규 계정을 등록하고 무료 체험(300달러 크레딧) 또는 무료 티어 조건을 확인합니다.
2.  **Compute Engine** > **VM 인스턴스** 메뉴에서 가상 서버 인스턴스를 하나 생성합니다.
    *   **OS**: Ubuntu 22.04 LTS (최신 x86/64 이미지)
    *   **권장 하드웨어 사양**: e2-standard-4 (4 vCPU, 16GB RAM) 이상을 권장합니다. (팰월드 전용 서버의 높은 메모리 소모 특성 때문입니다.)

### 2. GCP 가상 서버 방화벽(포트 개방) 규칙 구성
게임 클라이언트 접속을 허용하기 위해 인스턴스 네트워크 설정에 다음 규칙을 추가해야 합니다.
*   **게임 접속 포트**: `UDP 8211` (소스로부터의 IP 범위: `0.0.0.0/0` - 전체 개방)
*   **주의**: 보안을 위해 RCON 포트(`TCP 25575`) 및 REST API 포트(`TCP 8212`)는 구글 방화벽 상에서 외부로 노출되지 않도록 폐쇄하고 로컬에서만 통신하도록 설정하십시오.

### 3. 디스코드 개발자 포털 봇 생성 및 토큰 발급
1.  [Discord Developer Portal](https://discord.com/developers/applications)에 접속하여 **New Application**을 생성합니다.
2.  왼쪽 **Bot** 탭으로 이동하여 봇 계정을 생성하고, **Token**을 안전한 곳에 복사해 둡니다. (이 값이 `DISCORD_TOKEN`이 됩니다.)
3.  같은 Bot 탭 화면 하단의 **Privileged Gateway Intents** 섹션에서 다음 세 가지 필수 권한을 활성화합니다.
    *   `Presence Intent`
    *   `Server Members Intent`
    *   `Message Content Intent` (활성화하지 않을 시 봇의 슬래시 명령어가 차단될 수 있습니다.)
4.  왼쪽 **OAuth2** > **URL Generator** 탭으로 이동합니다.
    *   **Scopes**: `bot`, `applications.commands` 체크
    *   **Bot Permissions**: `Administrator` (또는 채널 읽기/쓰기, 역할 제어 권한 필수 체크)
5.  하단에 자동 생성된 URL 주소를 복사하여 브라우저에 접속한 뒤, 본인이 관리하는 디스코드 서버에 봇을 초대합니다.

### 4. MobaKeyGen (또는 puttygen.exe)으로 SSH 키 생성
1.  MobaXterm 상단 메뉴에서 **Tools** > **MobaKeyGen (SSH Key Generator)**을 실행합니다.
2.  하단의 Parameters 항목에서 **RSA**를 선택하고, **Generate** 버튼을 클릭합니다.
3.  진행 표시줄이 차오르는 동안 마우스를 무작위로 계속 움직여 임의의 난수를 생성합니다.
4.  키 생성이 완료되면 화면 중간의 **Key comment** 부분을 원하는 **`<사용자_계정명>`**으로 변경합니다. (예: `gcp-user` 등 영어 소문자로 구성된 계정명 권장)
5.  **Save private key** 버튼을 클릭하여 안전한 로컬 폴더에 비공개 키 파일(`.ppk`)을 저장합니다.
6.  맨 위 회색 텍스트 상자(`Public key for pasting into OpenSSH...`) 안에 있는 긴 문자열 전체를 복사합니다.

### 5. 구글 클라우드 콘솔에 공개 키 등록
1.  GCP 콘솔의 **Compute Engine** > **VM 인스턴스** 상세 페이지에서 상단의 **수정** 버튼을 클릭합니다.
2.  화면을 아래로 내려 **SSH 키** 항목을 찾은 후 **항목 추가**를 누릅니다.
3.  위에서 복사한 공개 키 전체를 입력란에 그대로 붙여넣습니다. (정상 등록 시 끝에 입력한 계정명이 감지됩니다.)
4.  맨 아래로 내려 **저장**을 누릅니다.

### 6. MobaXterm 세션 생성 및 최초 접속
1.  MobaXterm 왼쪽 위 **Session** 버튼을 누르고 **SSH** 세션을 추가합니다.
2.  **Remote host**: GCP에서 할당된 가상 서버의 **외부 IP 주소**를 입력합니다.
3.  **Specify username**: 키 생성 시 Key comment에 입력했던 **`<사용자_계정명>`**을 그대로 입력합니다.
4.  **Advanced SSH settings** 탭을 클릭한 뒤, **Use private key** 항목을 체크하고 저장해 두었던 비공개 키 파일(`.ppk`)을 등록합니다.
5.  **OK**를 누르면 가상 서버에 암호 입력 없이 안전하게 최초 접속됩니다.

### [단계 3] 자동 빌드 스크립트 실행
업로드가 완료되었다면 터미널 창에서 윈도우 스타일 줄바꿈 문자 정화 작업을 거친 뒤 설치 스크립트를 원클릭 구동합니다.

```bash
# 1. 윈도우 텍스트 편집기로 인한 에러(CRLF) 사전 자동 방지
sed -i 's/\r$//' setup.sh env .env

# 2. setup.sh 스크립트에 실행 권한 부여
chmod +x setup.sh

# 3. 자동 설치 스크립트 실행
./setup.sh
```

> **주의**: 팰월드 전용 서버 이미지를 최초로 다운로드하는 과정에서 약 **180초(3분)** 동안 대기 상태가 지속됩니다. 터미널 창을 닫지 마시고 설치 완료 메시지가 출력될 때까지 기다려 주십시오.

### [단계 4] 봇 계정에 Docker 권한 영구 등록 (필수)

## 🎯 유저용 디스코드 명령어 가이드

봇을 서버에 초대할 때 **`bot`**과 **`applications.commands`** 권한을 모두 체크하고 초대했다면 디스코드 대화창에서 슬래시( `/` )를 입력해 다음 관리 도구를 호출할 수 있습니다.

1.  **`/backup`**
    *   즉시 팰월드 게임에 저장을 명령하고, 가상 서버 로컬 드라이브에 안전하게 세이브 파일을 타르 압축하여 수동 백업을 생성합니다.
2.  **`/clean`**
    *   현황판 전용 채널에서 고정된 상태판 메시지 **하나만 제외하고** 유저들의 잡다한 채팅과 시스템 메시지 목록들을 일괄 정리합니다. (사용 전, 보존할 상태판 메시지를 디스코드에서 미리 **고정**해 주셔야 지워지지 않습니다.)
3.  **`/restart`**
    *   게임 내에 셧다운 방송을 송출하고 세이브를 저장한 뒤, **안전 백업을 수행하고** 도커 컨테이너를 리스타트합니다.
4.  **`/stop`**
    *   게임 서버를 디스코드 상에서 안전하게 정지합니다. (관리자 전용)
5.  **`/start`**
    *   정지된 게임 서버를 디스코드 상에서 다시 가동합니다. (관리자 전용)
6.  **`/디스크`**
    *   현재 서버의 디스크 총 용량, 사용 중인 용량, 사용률(%)을 실시간으로 조회하여 임베드 메시지로 출력합니다. (관리자 전용)
7.  **`/설정변경 <query>`**
    *   예: `/설정변경 query:EXP_RATE=2.0, CATCH_RATE=1.5`
    *   팰월드 서버의 `.env` 환경 변수 값을 한 번에 여러 개 변경하고 서버를 재시작합니다. 변경 전 이중 승인 절차를 거칩니다. (관리자 전용)
```
